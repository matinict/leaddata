# Unit-KidsLearn

> **"How can I build a unique kids learning channel where content changes automatically with topic, but visuals remain reusable and lightweight?"**

### ✅ My Constraints (Locked)
| Constraint               | My Choice |
|--------------------------|-------------|
| No heavy GPU / VRAM      | ✅ Lightweight only |
| No Sora/Runway/Pika      | ✅ No generative video |
| No manual asset creation | ✅ Fully automated |
| No generic ABC/animals   | ✅ Differentiated content |
| Reusable clip loops      | ✅ Fixed assets + dynamic composition |

### 🎯 My Real Goal
You're not building *kids animation* — you're building a **system-driven content factory for kids learning**, where:
- **The idea drives the content** (topic → script → structure)
- **Visuals stay predictable** (reusable clips, overlays, text animations)
- **Automation stays intact** (no manual intervention per video)

### 💡 Initial Direction (CF2-Compatible)
Given My rules, here's a viable path:

1. **Visual Layer = Static + Dynamic Text**
   - Reuse My existing clip loops (`intro/`, `p0/`, `c0/`, `mv/`, etc.)
   - Layer dynamic text overlays (definitions, facts, quizzes) generated by `Unit-Data`
   - Use `thumbnail_config` for AI-generated *still images only* (low cost) as background accents

2. **Content Layer = Topic-Driven Scripts**
   - `Unit-Data` generates kid-friendly scripts: "What is X?", "Why is it cool?", "Fun Fact!"
   - Output: `propose.md` → becomes scrolling text video via `debate_video.py` (repurposed as "learning video")

3. **Format Layer = Reusable Pipeline**
   - Same pipeline: `Data → Definition Video → Merge → Publish`
   - Just swap the *tone* and *structure* of the script, not the visual engine

4. **Differentiation = Idea Curation**
   - Use `scout_trend` to find *unexpected* kids-learning topics: "Why do clouds float?", "How do robots learn?", "What is code?"
   - My system's value = **curating + automating surprising educational content**, not generating new visuals
